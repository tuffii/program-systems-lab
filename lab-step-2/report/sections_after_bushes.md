## Табличное представление синтаксиса

Здесь размещаем синтаксис в формате таблиц, реализованных массивами структур языка С.

Низкоуровневый эквивалент синтаксиса, полученного после преобразования продукций в кусты, размещен в трех таблицах компилятора:

- `SINT` - таблица синтаксических правил в форме распознавания;
- `VXOD` - таблица входов в кусты грамматики;
- `TPR` - матрица непосредственной предшествуемости, на основе которой строится матрица достижимости.

Для персонального варианта N 11 были расширены размеры таблиц и буферов:

```c
#define NSINT     310
#define NVXOD      57
#define NSTROKA  4000
#define NNETRM     20
#define MAXLTXT   100
```

В таблице `SINT` изменены и добавлены ветви, которые реализуют:

- объявления `BIN FIXED` без явного указания разрядности;
- объявления `BIN FIXED INIT(...)` и `DEC FIXED INIT(...)` с десятичными литералами;
- десятичный целочисленный литерал `<LID>`;
- арифметическое умножение `*` как знак `<ZNK>`;
- логическое выражение `<AVL>` и логический оператор присваивания `<OPL>`;
- знак логической операции `<ZNL>`, которому соответствует терминал `=`.

```c
/*       AVI ... ZNK LID AVL OPL ZNL */
/*AVI*/  1  ...  0   0   1   0   0
/*CIF*/  0  ...  0   1   0   0   0
/*IPE*/  1  ...  0   0   0   1   0

```

Таким образом, в таблицах БД синтаксического анализатора символ `*` используется как арифметический знак `<ZNK>`, а символ `=` в правой части выражения используется как знак логической операции `<ZNL>`. Это соответствует высокоуровневому описанию:

```text
<ZNK> ::= + | - | *
<ZNL> ::= =
<AVL> ::= <AVI><ZNL><LIT> | <AVI><ZNL><LID> | <AVI><ZNL><IPE>
<OPL> ::= <IPE>=<AVL>;
```

## 4. Модификация алгоритма макета компилятора с ЯВУ

Модификация алгоритма макета компилятора с ЯВУ была произведена параметрически путем модификации той части параметров настройки компилятора ЯВУ, которая относится к семантике ЯВУ. Этой частью семантических параметров является библиотека семантических подпрограмм.

Объем модификаций этой библиотеки определяется объемом модификации синтаксиса ЯВУ. Для персонального варианта N 11 модификация синтаксиса привела к необходимости:

- изменить синтаксический анализатор для выбора ветви `<OPL>` при разборе оператора вида `C = A = C;`;
- изменить обработку объявлений в `ODC1()`;
- изменить генерацию арифметического выражения в `AVI2()`;
- добавить семантические подпрограммы для новых нетерминалов `LID`, `AVL`, `OPL`, `ZNL`;
- изменить завершение программы в `OEN2()`, чтобы формировались объявления двоичных и десятичных переменных, packed decimal констант, рабочего буфера и регистров.

Для выбора логического оператора присваивания добавлены две вспомогательные функции просмотра вперед. Они используются синтаксическим анализатором при выборе альтернативы:

```c
int is_logical_assignment ( int pos )
 {
  int eq_count = 0;
  int p = pos;

  while ( p < NSTROKA && STROKA [p] != '\x0' && STROKA [p] != ';' )
   {
    if ( STROKA [p] == '=' )
      eq_count++;
    p++;
   }

  return eq_count > 1;
 }

int has_equal_before_semicolon ( int pos )
 {
  int p = pos;

  while ( p < NSTROKA && STROKA [p] != '\x0' && STROKA [p] != ';' )
   {
    if ( STROKA [p] == '=' )
      return 1;
    p++;
   }

  return 0;
 }
```

В программе `sint_ANAL()` эти функции применяются в точке выбора альтернатив:

```c
if ( !strcmp ( SINT[J].DER, "OPA" ) &&
     SINT[J].ALT != 0 &&
     is_logical_assignment ( I ) )
  J = SINT[J].ALT;

if ( !strcmp ( SINT[J].DER, "AVI" ) &&
     SINT[J].ALT != 0 &&
     K > 0 &&
     !strcmp ( CEL[K-1].CEL1, "OPL" ) &&
     has_equal_before_semicolon ( I ) )
  J = SINT[J].ALT;
```

Данные изменения обеспечивают строгий маршрут разбора оператора `C = A = C;` через нетерминалы `<OPL>`, `<AVL>` и `<ZNL>`, а не через арифметический знак `<ZNK>`.

Семантическая подпрограмма `ODC1()` изменена так, чтобы заполнять таблицу имен `SYM` для двоичных и десятичных переменных, поддерживать отсутствие явной разрядности и сохранять десятичную инициализацию:

```c
int ODC1 ()
 {
  int i, cur;
  FORM ();
  if (IFORMT < 3) return 2;

  for ( i = 0; i < ISYM; i++ )
   if ( !strcmp ( SYM [i].NAME, FORMT [1] ) ) return 6;

  strcpy ( SYM [ISYM].NAME, FORMT [1] );

  if ( !strcmp ( FORMT [2], "BIN" ) ) SYM [ISYM].TYPE = 'B';
  else if ( !strcmp ( FORMT [2], "DEC" ) ) SYM [ISYM].TYPE = 'D';
  else return 2;

  if ( IFORMT > 4 && isdigit(FORMT[4][0]) )
   {
    strncpy ( SYM [ISYM].RAZR, FORMT [4], 4 );
    SYM [ISYM].RAZR[4] = '\0';
    cur = 5;
   }
  else
   {
    strcpy ( SYM [ISYM].RAZR, (SYM[ISYM].TYPE == 'B') ? "15" : "3" );
    cur = 4;
   }

  if ( cur < IFORMT && !strcmp ( FORMT [cur], "INIT" ) )
   {
    if (cur + 1 < IFORMT)
      strcpy ( SYM [ISYM].INIT, FORMT [cur + 1] );
    else
      SYM [ISYM].INIT[0] = '\0';
   }
  else
    SYM [ISYM].INIT[0] = '\0';

  ISYM++;
  return 0;
 }
```

В подпрограмме `AVI2()` добавлена генерация для:

- загрузки двоичной переменной в регистр `@R2`;
- копирования десятичной packed decimal переменной в буфер `@BUF`;
- умножения packed decimal значения на константу;
- преобразования packed decimal результата в двоичный регистр `@R1`.

## 5. Результаты экспериментальной проверки модифицированного компилятора с ЯВУ

Работоспособность модифицированного под персональный вариант макета компилятора с ЯВУ была подтверждена испытательным прогоном в среде ОС Linux. В качестве входного текста использована программа:

```pli
EX11: PROC OPTIONS ( MAIN );
DCL A BIN FIXED INIT ( 3 );
DCL B DEC FIXED INIT ( 3 );
DCL C BIN FIXED;
C = B * 5;
C = A = C;
END EX11;
```

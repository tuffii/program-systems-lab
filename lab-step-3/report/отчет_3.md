# Отчет по лабораторной работе: Разработка компилятора с Ассемблера

## 1. Описание примера из личного задания
**Формулировка личного задания:**
Необходимо выполнить доработку элементов макета учебной системы программирования до уровня, позволяющего обрабатывать «новые» для макета конструкции языка высокого уровня, примененные в варианте № 11.

**Исходный текст на ЯВУ (вариант № 11):**
```pli
EX11: PROC OPTIONS ( MAIN );
DCL A BIN FIXED INIT ( 3 );
DCL B DEC FIXED INIT ( 3 );
DCL C BIN FIXED;
C = B * 5;
C = A = C;
END EX11;
```

**Ассемблеровский эквивалент исходного текста контрольного примера (`examppl.ass`):**
```assembly
EXAMP    START 0            Start of program
         BALR  RBASE,0      Load base register
         USING *,RBASE      Set register as base
         L     RRAB,A       Load variable to register
         A     RRAB,B       Form intermediate value
         S     RRAB,C       Form intermediate value
         ST    RRAB,D       Form arithmetic expression value
         BCR   15,14        Exit from program
A        DC    F'3'         Variable definition
B        DC    F'4'         Variable definition
C        DC    F'5'         Variable definition
D        DC    F'0'         Variable definition
RBASE    EQU   15
RRAB     EQU   5
         END    EXAMP       End of program
```

**Ассемблеровский эквивалент личного задания (`ex11.ass`):**
```assembly
EX11     START 0            Init rel addr=0
         BALR  @RBASE,0     Set @RBASE=cur PC
         USING *,@RBASE     Use @RBASE for addr
         ZAP   @BUF(8),B(3) Copy pdec B->BUF
         MP    @BUF(8),@FIVE(1) Mul BUF by 5->BUF
         CVB   @R1,@BUF     pdec BUF ->int @R1
         STH   @R1,C        Store int ->C
         LH    @R2,A        Load A ->@R2
         CH    @R2,C        Compare @R2 with C
         BC    8,@TRUE      Branch if equal
         LA    @R3,0        Set @R3=0 (F)
         B     @SAVE        Jump to save
@TRUE    LA    @R3,1        Set @R3=1 (T)
@SAVE    STH   @R3,C        Store bool ->C
         BCR   15,@RVIX     Return to caller
A        DC    H'3'         Store halfword init
B        DC    PL3'3'       Store pdec init
C        DS    H            Reserve halfword
@FIVE    DC    PL1'5'       Store packed const
         DS    0F           Align next item
@BUF     DC    PL8'0'       Clear packed BUF
@RBASE   EQU   5            Set @RBASE=5
@R1      EQU   6            Set @R1=6
@R2      EQU   2            Set @R2=2
@R3      EQU   3            Set @R3=3
@RVIX    EQU   14           Set @RVIX=14
         END                Program ends
```

**Множество языковых конструкций, отсутствующих в ассемблеровском эквиваленте контрольного примера:**
При сравнении `examppl.ass` и `ex11.ass` выявлены следующие новые языковые конструкции (машинные команды и псевдооперации):
1. Машинные команды формата RX:
   - `CVB` (Convert to Binary)
   - `STH` (Store Halfword)
   - `LH` (Load Halfword)
   - `CH` (Compare Halfword)
   - `BC` (Branch on Condition)
   - `LA` (Load Address)
   - `B` (Branch Unconditional — расширенная мнемоника для команды безусловного перехода)
2. Машинные команды формата SS:
   - `ZAP` (Zero and Add Packed)
   - `MP` (Multiply Packed)
3. Псевдооперации и новые типы данных для `DC` и `DS`:
   - `H` (Halfword) для `DC` и `DS`
   - `PLn` (Packed Decimal) для `DC` с указанием длины (например, `PL3`, `PL1`, `PL8`)
   - `0F` для выравнивания в `DS` (выравнивание на границу полного слова)

## 2. Постановка задачи модификации программы-макета компилятора с АССЕМБЛЕРА
Для обеспечения компиляции ассемблеровского эквивалента примера из личного задания необходимо модифицировать программу-макет компилятора с АССЕМБЛЕРА. 
Выбранный способ модификации:
1. **Расширение таблицы машинных операций (TMOP):** добавление записей для новых машинных команд `CVB`, `STH`, `LH`, `CH`, `BC`, `LA`, `B`, `ZAP`, `MP` с указанием их мнемоник, кодов операций (КОП) и форматов (RX, SS).
2. **Доработка функциональных блоков обработки машинных команд:**
   - Добавление поддержки команд формата SS (в макете изначально поддерживались только RR и RX). Для этого потребуется создать новый функциональный блок обработки команд типа SS, который будет парсить операнды с указанием длины (например, `D1(L1),D2(L2)`).
   - Расширение блока обработки команд типа RX для поддержки мнемоники `B` (ее можно транслировать как `BC 15,`).
3. **Доработка функциональных блоков обработки псевдоопераций:**
   - Модификация блока обработки псевдооперации `DC` для поддержки типов данных `H` (Halfword, 2 байта) и `P` (Packed Decimal, переменная длина).
   - Модификация блока обработки псевдооперации `DS` для поддержки типа `H` и выравнивания `0F`.

## 3. Перечень БД программы-макета компилятора с АССЕМБЛЕРА
В основу функционирования макета компилятора с Ассемблера положен алгоритм, использующий в своей работе базу данных (БД), включающую в себя следующие элементы:
- **Исходный текст** (массив строк, содержащий текст на языке Ассемблера).
- **Счетчик адреса (Сч.Адр.)** — используется для отслеживания текущего относительного адреса при генерации объектного кода.
- **Таблица символов (TSYM)** — хранит имена меток и переменных с их относительными адресами.
- **Таблица псевдоопераций (TPOP)** — содержит перечень поддерживаемых псевдоопераций (START, DC, DS, USING, EQU, END).
- **Таблица машинных операций (TMOP)** — содержит перечень поддерживаемых машинных команд, их КОП и типы (RR, RX, SS).
- **Таблица базовых адресов (TBASE)** — хранит информацию о базовых регистрах, объявленных через USING.
- **Двоичный эквивалент исходного текста** — формируемый набор объектных карт (ESD, TXT, END).

## 4. Перечень и пояснения модификаций алгоритма программы-макета компилятора с АССЕМБЛЕРА
Для реализации поставленной задачи в алгоритм программы-макета вносятся следующие изменения:

1. **Модификация таблицы TMOP:**
   В таблицу `TMOP` добавляются новые строки:
   - `{"CVB", 0x4F, 'R', 'X'}`
   - `{"STH", 0x40, 'R', 'X'}`
   - `{"LH",  0x48, 'R', 'X'}`
   - `{"CH",  0x49, 'R', 'X'}`
   - `{"BC",  0x47, 'R', 'X'}`
   - `{"LA",  0x41, 'R', 'X'}`
   - `{"B",   0x47, 'R', 'X'}` (с обработкой маски условия 15)
   - `{"ZAP", 0xF8, 'S', 'S'}`
   - `{"MP",  0xFC, 'S', 'S'}`

2. **Добавление блока обработки команд типа SS:**
   Создается новая функция для разбора операндов команд формата SS: `OP1(L1),OP2(L2)`. Алгоритм должен извлекать базовые регистры, смещения и длины операндов, после чего формировать 6-байтовый машинный код.

3. **Модификация блока обработки `DC` (Define Constant):**
   - Добавляется распознавание типа `H` (Halfword). Значение константы преобразуется в 2-байтовое двоичное представление.
   - Добавляется распознавание типа `P` (Packed Decimal) с модификатором длины `L` (например, `PL3`). Значение константы переводится в упакованный десятичный формат (каждая цифра занимает полубайт, последний полубайт — знак) и занимает указанное количество байт.

4. **Модификация блока обработки `DS` (Define Storage):**
   - Добавляется распознавание типа `H` (увеличение счетчика адреса на 2).
   - Добавляется поддержка `0F` (выравнивание счетчика адреса на границу полного слова без выделения памяти).

5. **Модификация блока обработки `EQU` (Equate):**
   - Убедиться, что `EQU` корректно обрабатывает символические имена регистров (например, `@R1`, `@RBASE`), которые используются в операндах команд.

## 5. Выводы по результатам работы
В ходе выполнения работы был проведен анализ исходного ассемблеровского эквивалента контрольного примера и ассемблеровского эквивалента личного варианта № 11. Выявлены новые языковые конструкции, требующие поддержки со стороны макетного компилятора с Ассемблера.

**Позитивные моменты:**
- Структурированность программы-макета (наличие таблиц TMOP, TPOP и отдельных блоков обработки) позволяет легко расширять функционал компилятора.
- Большинство новых команд (CVB, STH, LH, CH, BC, LA) относятся к уже поддерживаемому формату RX, что требует лишь добавления записей в таблицу TMOP.

**Негативные моменты/Сложности:**
- Появление команд формата SS (ZAP, MP) требует разработки нового алгоритма разбора операндов с длинами, что усложняет лексический и синтаксический анализ строки ассемблера.
- Реализация упаковки десятичных чисел (Packed Decimal) для псевдооперации `DC PLn` требует аккуратной работы с битовыми операциями и полубайтами.

**Общая оценка:**
Поставленная задача декомпозирована, определены конкретные точки модификации БД и алгоритма компилятора. Спроектированный перечень модификаций является достаточным для обеспечения успешной трансляции ассемблеровского эквивалента личного задания в объектный код.
